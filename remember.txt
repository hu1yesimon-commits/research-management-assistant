职责
  1. DeDuplicator 输出去重后的 PaperMetadata
  2. LLMJudge 接收 PaperMetadata + query
  3. LLMJudge 让模型根据 title / abstract / venue / venue_type / published_date 判断 llm_relevance_score 和 quality_score
  4. embedding_relevance_score 最好由单独的 embedding 服务提前算好，再传给 LLMJudge
  5. novelty_score 用规则函数算好，不让 LLM 主观生成
  6. LLMJudge 调用 calculate_final_score()
  7. 最后用 sort_by_final_score() 降序排列结果


Langgraph 化
  阶段 1：普通 Python 服务

  PaperSearchService + DeDuplicator + LLMJudge

  阶段 2：简单 LangGraph 编排

  search → dedup → judge → register

  阶段 3：加人工确认

  judge 后 uncertain 的论文暂停，让你手动确认

  阶段 4：加持久化

  用 checkpointer 保存流程状态，避免中途失败全部重跑

quality_score 可以先简单做
    relevance_score
    quality_score
    final_score = 0.65 * relevance_score + 0.25 * quality_score + 0.10 * freshness_score
    
    relevance_score：LLM 根据 query / research profile / abstract 判断

    quality_score：citation_count、venue 是否存在、是否有 DOI、是否有 PDF、是否来自可信源

    freshness_score：发布日期越近越高
